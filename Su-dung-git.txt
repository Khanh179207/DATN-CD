⚡️ QUY TRÌNH GIT (GOMET TEAM)
🛑 NGUYÊN TẮC VÀNG
*KHÔNG code trực tiếp trên main hoặc develop.

*LUÔN tạo nhánh mới (feature/...) khi bắt đầu làm việc.

*LUÔN Pull code về trước khi Push.

1️⃣ BẮT ĐẦU code TÍNH NĂNG MỚI 
git checkout develop (chuyển về nhánh chung của cả nhóm, nơi có tất cả code của mọi người)       
git pull origin develop (lấy code từ nhánh dev về máy tính của mình)       
git checkout -b feature/ten-chuc-nang (tạo 1 không gian làm viên cá nhân của mình để code chức năng)

2️⃣ LẤY CODE MỚI VỀ KHI CÓ THÀNH VIÊN KHÁC VỪA PUSH LÊN DEVELOP (chỉ làm sau khi đã làm bước  1)
Chọn 1 trong 2 trường hợp sau:

✅ TH1: Chưa code gì cả (Máy sạch)

git pull origin develop (lấy code từ nhánh dev về máy tính của mình)

⚠️ TH2: Đang code dở (Đã sửa file)

git stash (Cất gọn code đang viết dở vào kho tạm, trả lại máy sạch)
git pull origin develop (cập nhật code mới từ develop)
git stash pop (Lấy code cất trong kho ra ráp lại vào file)
                  
# -> Nếu báo Conflict (đỏ): Mở file sửa lỗi -> Lưu -> git add . -> git commit

3️⃣ Khi code xong (PUSH & PR)

git add .  (thêm các file đã code vào git trên máy)
git commit -m "Hoan thien chuc nang ABC"  (cam kết các file này là gì ) 
git push origin feature/ten-chuc-nang ( đẩy code của mình lên server github)

👉 Sau đó: Lên GitHub -> Tạo Pull Request vào develop.
